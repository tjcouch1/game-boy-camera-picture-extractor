# Frame feature — design

Status: approved
Branch: `add-frames`

## Goal

Let users wrap their extracted Game Boy Camera image inside a frame chosen from
official Game Boy Camera frame sheets. The frame becomes part of the rendered
preview, the downloaded PNG, the shared image, and the clipboard image. Users
choose a default frame globally and can override per-result.

## Constraints set by the user

- Frame sheet PNGs live canonically under `supporting-materials/frames/`. A
  build script copies image files only (no `.md`) into
  `packages/gbcam-extract-web/public/frames/`. The copied folder is
  `.gitignore`-d at the repo root, mirroring the
  `public/instructions-assets/` pattern.
- The pure splitter / composer logic ships in `gbcam-extract` so it is
  testable in Node and reusable by the web package. Tests live in
  `gbcam-extract`.
- Each frame's hole is exactly 128 × 112 — same size as the camera image — so
  composition is a plain copy at `(holeX, holeY)`. No mask required.
- Frame pixels are already in (or extremely close to) the four GB grayscale
  values `{0, 82, 165, 255}`. The splitter snaps each non-background pixel to
  the nearest of those four; no perceptual quantization is needed.
- If composition fails for any reason, the renderer falls back to the
  palette-applied bare image. The image is the important part.
- The list of frame-sheet URLs is generated by the build script — no
  hard-coded URL list in the hook.
- Frame selection persisted to localStorage stores only the frame `id`, not
  the pixel data.
- Persisted-data changes must be **backward compatible**: existing
  `gbcam-current-results`, `gbcam-image-history`, and `gbcam-app-settings`
  entries continue to load and behave the same.
- Dedup tiebreaker: alphabetical sheet stem (so `Frames_JPN` wins over
  `Frames_USA` for identical frames).
- Frame ID format: `<sheetStem>:<type>:<index>` (e.g. `Frames_USA:normal:1`).
  Used only for save/load — not surfaced in filenames, not displayed to the
  user.

## `gbcam-extract` — frame splitter

### Algorithm

1. Read top-left pixel → `bgColor`. Match with ±2 per-channel tolerance to
   forgive non-PNG-perfect inputs.
2. Build a `W × H` binary mask: `1` for non-background pixels, `0` for
   background.
3. Build a 2D prefix-sum (integral image) of the **background** mask. Used to
   answer "is the 128 × 112 rectangle starting at `(x, y)` entirely
   background?" in O(1).
4. Find connected components (4-connectivity flood fill) of the non-background
   mask. Each component becomes a candidate frame, with bounding box
   `(x0, y0, w, h)`.
5. Filter candidates:
   - Drop if `w < 32` or `h < 32` (filters out the
     "Normal frames" / "Wild frames" labels and credits text).
   - For each surviving candidate, scan top-to-bottom, left-to-right inside
     the bounding box for the first `(holeX, holeY)` such that the
     128 × 112 rectangle there is entirely background. Use the prefix sum.
   - If no such rectangle exists, drop the candidate (it's artwork without a
     hole — credits, etc).
6. Type:
   - `"normal"` if `w === 160 && h === 144`.
   - `"wild"` otherwise.
7. Sort surviving candidates by `(y0, x0)` ascending → stable
   left-to-right, top-to-bottom indexing within each type. Number per-type
   from 1.
8. For each frame:
   - Allocate `Uint8ClampedArray(w × h)`.
   - For each pixel in the frame's bounding box: if it is background, write
     `255` (so the hole renders as the lightest palette colour when the
     frame is shown alone in the picker). Otherwise, take the RGB → grayscale
     via the R channel (frames are already grayscale; using R is exact
     enough). Snap that grayscale to the nearest of `{0, 82, 165, 255}`.
   - Record `holeX`, `holeY` relative to the frame's top-left.

### Output type

```ts
// gbcam-extract/src/frames/types.ts
export interface Frame {
  /** Stable identifier: "<sheetStem>:<type>:<index>" */
  id: string;
  sheetStem: string;
  type: "normal" | "wild";
  /** 1-based index, scoped to (sheetStem, type) */
  index: number;
  width: number;
  height: number;
  /** length = width × height; values in {0, 82, 165, 255} */
  pixels: Uint8ClampedArray;
  /** top-left of the 128 × 112 hole, in frame-local coords */
  holeX: number;
  holeY: number;
}

export function splitSheet(
  sheet: GBImageData,    // RGBA
  sheetStem: string,
): Frame[];
```

### Dedup across sheets

```ts
export function dedupeFrames(frames: Frame[]): Frame[];
```

- Hashes `(width, height, type, pixels)` for each frame (FNV-1a or similar
  on the byte stream — exact dedup, not perceptual).
- Sorts the input by `sheetStem` ascending before passing through, so
  alphabetically earlier stems win.
- `holeX`/`holeY`/`id` are not part of the hash because two identical-pixel
  frames have identical hole positions by definition.
- Output preserves the original arrival order of unique frames.

## `gbcam-extract` — frame composer

```ts
// gbcam-extract/src/frames/compose.ts
export function composeFrame(
  image: GBImageData,                            // 128×112 quantized grayscale
  frame: Frame,
  palette: [string, string, string, string],
): GBImageData;                                  // RGBA
```

- Allocates `frame.width × frame.height × 4` bytes.
- Writes the frame's pixel grid into a temporary `GBImageData` whose
  values are already in `{0, 82, 165, 255}`.
- Copies image pixels (raw grayscale values) over the
  `frame.holeX, frame.holeY → +128, +112` region.
- Calls `applyPalette(combined, palette)` to produce the RGBA output.

### Failure mode

If `image`, `frame`, or `palette` is malformed, `composeFrame` throws. The
**caller** is responsible for catching and falling back to
`applyPalette(image, palette)` — a thin one-line wrapper in
`buildOutputCanvas()` and the result-card preview effect. We keep the pure
function honest (no silent degradation) and put the policy at the call
site.

## Web data layer

### `scripts/copy-frames.ts`

- Walks `supporting-materials/frames/**/*.{png,jpg,jpeg,gif,webp}`. Skips
  `.md`, `.txt`, and any other non-image extension.
- Copies into `packages/gbcam-extract-web/public/frames/`, preserving
  subfolder structure (so
  `the-spriters-resource/Frames_USA.png` lands at
  `public/frames/the-spriters-resource/Frames_USA.png`).
- Writes `packages/gbcam-extract-web/src/generated/FrameSheets.ts`:

```ts
// auto-generated — do not edit
export interface FrameSheetEntry { url: string; stem: string; }
export const FRAME_SHEETS: ReadonlyArray<FrameSheetEntry> = [
  { url: "./frames/the-spriters-resource/Frames_JPN.png", stem: "Frames_JPN" },
  { url: "./frames/the-spriters-resource/Frames_USA.png", stem: "Frames_USA" },
];
```

- Sorts the array by `stem` ascending so dedup tiebreaking is implicit.

### `package.json` wiring (web)

Hook into `dev`, `dev:host`, `build`, `preview`, `preview:host`,
`postinstall` — the same scripts that already invoke
`build:instructions`. Add a `build:frames` script and chain it before
`build:instructions`.

### Root `.gitignore` additions

```
# Generated frame sheets and copied images
packages/gbcam-extract-web/src/generated/FrameSheets.ts
packages/gbcam-extract-web/public/frames
```

### `useFrameCatalog` hook

`packages/gbcam-extract-web/src/hooks/useFrameCatalog.ts`:

- On first call, fetch each entry in `FRAME_SHEETS`, decode via Canvas
  (mirroring the `fileToGBImageData` pattern in `useProcessing`), call
  `splitSheet`, concat, then `dedupeFrames`.
- Caches the resulting `Frame[]` in a module-level singleton so remounts
  don't re-process.
- Returns `{ status: "loading" | "ready" | "error", frames: Frame[],
  error?: string, getFrameById(id): Frame | undefined }`.
- No localStorage caching — the work is small (~few hundred ms,
  one-time per page load).

## UI

### `<FramePicker>`

`packages/gbcam-extract-web/src/components/FramePicker.tsx`. Built on shadcn
`Popover` + `Button` + lucide icons.

Props:

```ts
interface FramePickerProps {
  value: FrameSelection;
  onChange: (next: FrameSelection) => void;
  palette: [string, string, string, string];
  frames: Frame[];
  /** "default" omits the "Default" tile (used in the global default picker). */
  /** "result"  includes a "Default — <currently: …>" tile at the very top.   */
  mode: "default" | "result";
  /** Default frame name (the one currently selected globally), shown on the */
  /** "Default — …" tile in result mode. Optional in default mode.            */
  defaultFrameLabel?: string;
}
```

Trigger:

- A small `Button` containing:
  - 16–20 px square `<canvas>` rendering the current selection's
    thumbnail at the active palette. For "no frame" / "default"
    fallbacks: a lucide `<Frame>` icon.
  - The selection's display name: `"No frame"`, `"Default — <name>"`, or
    the frame's display name.
  - A `<ChevronDown>` end-icon (`data-icon="inline-end"`).

Popover content (~600 px on desktop, full-viewport-width on mobile):

- Pinned tiles at top:
  - In `mode="result"`: a "Default — currently <name>" tile.
  - Always: a "No frame" tile.
- `<h4>Normal frames</h4>` → CSS grid; each tile is a 1× canvas at native
  160 × 144 plus a name underneath. ~2 cols mobile, 3 cols desktop.
- `<h4>Wild frames</h4>` → grid; tiles are native size, but capped to fit
  column width (`max-width: 100%`). Same column count.
- Click selects and closes the popover. No search, no filter.

All thumbnails render with the current palette via `composeFrame` (no image
provided — the hole renders as the lightest palette color via the `255` in
`frame.pixels`).

Display name for a frame: `"<sheetStem> — <type> #<index>"`, e.g.
`"Frames_JPN — normal #3"`. Plain and unambiguous, since users have no
canonical names for these frames.

### `FrameSelection` type

```ts
export type FrameSelection =
  | { kind: "default" }                  // result-card only — follow global default
  | { kind: "none" }                     // explicit "no frame"
  | { kind: "frame"; id: string };
```

- Stored in JSON-friendly form.
- Global default may only be `none` or `frame` (no `default`).
- Per-result override may be any of the three; `default` is the initial
  state for new results.

### Default-frame setting

- Add `defaultFrame?: FrameSelection` to `AppSettings` in
  `useAppSettings.ts`. Default `{ kind: "none" }`.
- In `App.tsx`, render the default `<FramePicker mode="default" …>`
  inside the existing results bar after Preview Scale (option A —
  invisible until results exist, same as the scale controls).

### `ResultCard` layout

- Add props `frameOverride: FrameSelection`, `onFrameOverrideChange`,
  `effectiveFrame: Frame | null`, `frames: Frame[]` (the catalog,
  for the picker), `defaultFrameLabel: string`.
- New layout (CardContent):
  - Mobile (`flex-col`): canvas first (`order-1`), then a column with
    `[buttons row, frame picker]` (`order-2`).
  - Desktop (`sm:flex-row`): left column is `[buttons row, frame picker]`
    (`sm:order-1`); right column is the canvas (`sm:order-2`).
- Effective frame resolution lives in `App.tsx`:

```ts
const effective =
  override.kind === "default"
    ? defaultFrame
    : override;
const effectiveFrame = effective.kind === "frame"
  ? catalog.getFrameById(effective.id) ?? null
  : null;
```

  The card receives `effectiveFrame` already resolved.
- The preview canvas effect uses
  `effectiveFrame ? composeFrame(grayscale, effectiveFrame, palette)
                  : applyPalette(grayscale, palette)`.
  Wrapped in `try/catch`; on failure falls back to the bare image.
- Canvas dimensions become
  `(effectiveFrame?.width ?? 128) * previewScale ×
   (effectiveFrame?.height ?? 112) * previewScale`.

### Download / share / copy

- `buildOutputCanvas` takes `effectiveFrame` and uses the same
  compose-or-fallback path. Output dimensions track the frame.
- Filename remains `<base>_<paletteSlug>_gb.png`. Frame name not added to
  filename — keep filenames short and stable.

### Per-result override storage

`ProcessingResult` gains:

```ts
frameOverride?: FrameSelection;   // undefined = { kind: "default" }
```

- Lives in `useProcessing.ts` (current results) and `useImageHistory.ts`
  (history). Both already serialize `ProcessingResult` to localStorage —
  the new optional field round-trips by default.
- **Backward compatibility:** when deserializing, results lacking
  `frameOverride` default to `{ kind: "default" }` (which means "follow
  global default", which itself defaults to "no frame"). Old data
  therefore renders identically to before.
- The override stores only the frame `id` (string). Resolution to the
  full `Frame` object happens at render time via the catalog.

## Tests (gbcam-extract, vitest)

`packages/gbcam-extract/tests/frames/`:

### `split-sheet.test.ts`

- Loads `Frames_USA.png` via the existing `tests/helpers/load-image.ts`
  (`sharp`).
- Asserts:
  - Total frame count is `> 0` and matches the snapshotted number.
  - Counts of `normal` vs `wild` match snapshot.
  - Every frame has `width > 0`, `height > 0`, hole at valid position
    (`holeX + 128 ≤ width`, `holeY + 112 ≤ height`).
  - All frames of type `"normal"` have exactly `160 × 144` dimensions.
  - All frames of type `"wild"` have non-`160 × 144` dimensions.
  - Every pixel value is in `{0, 82, 165, 255}`.
  - IDs are unique and follow the `<stem>:<type>:<index>` shape.
  - Indices are 1-based and contiguous within `(stem, type)`.
- Same assertions for `Frames_JPN.png`.

### `dedupe.test.ts`

- Splits both sheets, dedupes, asserts:
  - Result count `<` sum of both inputs.
  - Among any duplicate set, the `Frames_JPN` ID is the survivor
    (alphabetical tiebreaker).
  - Single-sheet input passes through unchanged.
  - Empty input returns `[]`.

### `compose.test.ts`

- Synthetic case: build a 160×144 frame with a 128×112 hole at `(16, 16)`,
  pixels filled with grayscale value `0` outside the hole and `255` inside.
  Compose with a 128×112 image filled with grayscale value `82` and the
  default palette `["#FFFFA5", "#FF9494", "#9494FF", "#000000"]`.
  Assert:
  - Output dims are `160 × 144`.
  - Hole region pixels match `applyPalette` of value `82` → `#9494FF`.
  - Frame region pixels match `applyPalette` of value `0` → `#000000`.
- Real-frame case: pick the first frame from `Frames_USA`, compose with a
  128×112 image of value `255`, assert hole region matches the lightest
  palette color and frame region equals `applyPalette` of `frame.pixels`.

No web-package tests — current `gbcam-extract-web` has no test
infrastructure; standing one up is out of scope for this feature.

## Out of scope

- Adding more frame sheets beyond the two Spriters Resource ones. The
  build script handles arbitrary additions automatically — drop a PNG
  into `supporting-materials/frames/` and rerun `dev` / `build`.
- A web-side test suite.
- Custom user-uploaded frames.
- Per-frame display names beyond `<stem> — <type> #<index>`.
- Storing the picker's chosen frame in any backend / cloud sync.
